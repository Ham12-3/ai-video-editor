# ClipForge

AI-powered video editor for short-form content (TikTok, Instagram Reels, YouTube Shorts). Upload a video, describe the edits in plain English, and ClipForge analyses the footage with GPT-5.4, transcribes with Whisper, generates literal on-topic illustrations with Google's Nano Banana 2, and renders the final cut with FFmpeg. You bring your own API keys, so you control the costs.

## What It Does

1. **Upload** a video (MP4, MOV, WebM, AVI up to 500MB)
2. **Describe** the edit in natural language: "Trim dead air, remove silences and filler words, speed up to 1.2x, reframe to 9:16, add karaoke captions, drop in illustrations of whatever I mention"
3. **AI analyses** the video: extracts audio, transcribes with Whisper + gpt-4o-mini-transcribe, detects silences and filler words, samples keyframes, sends everything to **GPT-5.4** for understanding
4. **Review** the edit plan (EDL): see what the AI decided, toggle operations on or off, tweak caption styling, refine with follow-up prompts
5. **Render** with FFmpeg: cuts applied, captions burned in, vertical reframe, audio normalised to -14 LUFS, optional illustration overlays generated by **Nano Banana 2** (Gemini 3.1 Flash Image)
6. **Download** or iterate

## Recent Changes

- **Switched illustration generation from DALL-E 3 to Google Nano Banana 2.** Much better at literal, on-topic imagery for video overlays. Adds a third API key slot in Settings.
- **Bumped analysis model to GPT-5.4** (was GPT-4.1). Self-review uses GPT-5.4-mini.
- **Fixed render pipeline drop-all-frames bug.** The filter chain was renumbering PTS at the end instead of right after `select`, which caused FFmpeg to drop every video frame and produce audio-only output. Now ordered correctly. See [`src/lib/video/ffmpeg.ts`](src/lib/video/ffmpeg.ts).
- **Deduplicated stacked operations.** When GPT emitted multiple `speed` / `caption` / `reframe` / `silence_remove` ops, the renderer was applying each one stacked, mangling the timeline. Now applied at most once per render. Also merges multiple `illustration` ops into a single overlay pass.
- **Forced libx264 when select filter is in the chain.** Intel QSV throws "Current frame rate is unsupported" on the variable frame rate that select+setpts produces.
- **Fixed stuck rendering spinner.** UI now respects terminal DB states (`failed`, `completed`) when the SSE error event is missed.
- **Tightened the GPT prompt** to forbid duplicate global ops and to plan 6-12 illustrations of literal subjects (not abstract concepts).

## Setup

### 1. Install dependencies

```bash
cd clipforge
npm install
```

### 2. Start databases

```bash
docker compose up -d
```

This starts PostgreSQL 16 and Redis 7 with health checks and persistent volumes.

### 3. Configure environment

```bash
cp .env.example .env.local
```

Generate an encryption key:

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

Paste the output as `ENCRYPTION_KEY` in `.env.local`. The other defaults work for local dev.

### 4. Push database schema

```bash
DATABASE_URL="postgresql://user:password@localhost:5432/clipforge" npm run db:push
```

### 5. Start the dev server

```bash
npm run dev
```

Open **http://localhost:3000**.

### 6. First run

1. Sign up at `/sign-up` with email and password
2. Go to **Settings** and add API keys:
   - **OpenAI** (`sk-...`) — required for analysis and transcription. Get one at [platform.openai.com/api-keys](https://platform.openai.com/api-keys).
   - **Google Gemini** (`AIza...`) — required for illustration overlays. Get one free at [aistudio.google.com/apikey](https://aistudio.google.com/apikey).
   - **Anthropic** (`sk-ant-...`) — optional, alternative for video analysis with Claude.
3. Click **New Project**, upload a video
4. Type a prompt and click **Process Video**
5. Review the AI edit plan, toggle anything you don't want, then click **Render Video**
6. Download the result or iterate

## AI Pipeline

The analysis pipeline runs these steps in order, streaming progress via SSE:

1. **Audio extraction**: FFmpeg extracts 16kHz mono WAV
2. **Silence detection**: FFmpeg `silencedetect` filter finds gaps over 0.5s
3. **Transcription**: OpenAI Whisper for word-level timestamps, optionally combined with `gpt-4o-mini-transcribe` for cleaner text (hybrid mode)
4. **Filler word detection**: Scans the transcript for "um", "uh", "like", "you know", etc.
5. **Frame extraction**: Scene-change detection (fallback: 1 frame per 2s), capped at 30 frames
6. **Pass 1 — transcript analysis**: GPT-5.4 reads the transcript, picks the hook moment, identifies topic segments, flags weak content
7. **Pass 2 — visual analysis**: GPT-5.4 receives keyframes plus pass 1's structured analysis plus the user's prompt, returns a full Edit Decision List in structured JSON
8. **Self-review**: GPT-5.4-mini double-checks the EDL for sentence-mid cuts, contradictions, and broken timing. Confidence below 0.7 returns the corrected version.

The EDL supports these operations:

| Op | What it does |
|----|--------------|
| `trim` | Keep a single time range (start, end) |
| `cut` | Remove specific segments |
| `caption` | Burn in subtitles (karaoke / word-by-word / sentence / minimal), with full styling control |
| `speed` | Change playback speed for segments, optionally preserving pitch |
| `silence_remove` | Cut silent gaps and named filler words using pre-detected timestamps |
| `reframe` | Aspect ratio change (9:16, 1:1, 4:5) with centre-crop |
| `transition` | Crossfade / cut / swipe / zoom (planned) |
| `illustration` | Overlay AI-generated images at specific timestamps. Each illustration gets a literal Nano Banana 2 picture grounded in the transcript text. |

## Rendering Pipeline

The render pipeline turns the EDL into FFmpeg commands:

1. **Subtitle generation**: ASS format with karaoke `\kf` tags, Arial Black font, thick outline, positioned at three-quarters screen height for mobile readability
2. **FFmpeg command building**: trim via `-ss`/`-to`, cuts via `select` + `aselect` with PTS renumbered immediately to keep frames monotonic, then speed (`setpts=1/x*PTS`, `atempo`), then crop, then scale, then subtitle burn-in. Audio gets EBU R128 loudness normalisation (-14 LUFS) and 0.5s fade in/out.
3. **Encoder selection**: hardware (NVENC / AMF / QSV) when available, automatic libx264 fallback when the chain contains `select` (Intel QSV does not handle the resulting variable frame rate)
4. **Execution**: progress parsed from stderr `time=` output, streamed via SSE
5. **Output verification**: `ffprobe` confirms the output actually has a video stream. If not, the renderer throws a clear error rather than handing back a silently broken file.
6. **Illustration overlays** (separate pass): Nano Banana 2 generates one image per illustration moment, FFmpeg overlays them with `filter_complex`, position rotated across corners, opacity 0.9
7. **Output**: H.264 / AAC MP4 with `+faststart` for web streaming

## API Key Security

Users bring their own keys for OpenAI, Gemini, and optionally Anthropic. Keys are:

- Encrypted with AES-256-GCM before storage (key from env var, random IV per record, auth tag for tamper detection)
- Decrypted only server-side when making outbound API calls
- Never sent to the browser, exposed in logs, or included in API responses
- Deletable per-provider at any time from the Settings page
- The encryption module imports the `server-only` package, so it cannot be bundled into client code

The detailed threat model is in [ARCHITECTURE.md](ARCHITECTURE.md).

## Cost Model

All AI costs are charged to your own OpenAI / Gemini account. Before each analysis, ClipForge shows estimated tokens, frame count, and dollar cost in a confirm dialog.

**Approximate costs per video** (60s talking-head clip):

| Step | Model | Cost |
|------|-------|------|
| Transcription | Whisper | ~$0.006 |
| Analysis (pass 1 + 2) | GPT-5.4 | ~$0.05 |
| Self-review | GPT-5.4-mini | ~$0.005 |
| Illustrations (10 images) | Nano Banana 2 | ~$0.45 |
| **Total with illustrations** | | **~$0.51** |
| Total without illustrations | | ~$0.06 |

If no Gemini key is configured the illustration step is skipped, the render still works, and you only pay for analysis and transcription.
